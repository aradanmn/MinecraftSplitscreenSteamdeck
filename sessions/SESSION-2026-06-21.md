## RAW TRANSCRIPT PUSH 7 — 2026-06-21 (continued session)

### Problem: Orchestrator blocked, T1.3/T1.4 failing

The core bug causing T1.3 (slot inactive within 30s of kill) and T1.4 (PID reaped)
to always fail was identified and fixed this session.

ROOT CAUSE: `spawn_instance 2>&1 | sed 's/^/[orchestrator] /' >&2`
The sed pipe keeps its write-end open as long as ANY process inherits it — including
bwrap, PolyMC, and Java (descendants of spawn_instance's setsid). So the orchestrator
was blocked in that pipe for the entire lifetime of Minecraft. It couldn't read
`SLOT_DIED` from the FIFO until AFTER Minecraft was already dead — defeating T1.3's
30-second window.

Prior fix (375693b): Redirected bwrap's stdout/stderr to $LOG. This helped but the
orchestrator was STILL in the pipe because spawn_instance itself (doing _poll_for_java,
_poll_for_window, sync_apply_layout) continued running inside the pipe for ~25-35 more
seconds. Those polling steps ran inside spawn_instance's shell which held the pipe
write-end — regardless of where bwrap output was going.

THE FIX (1b06f5c — modules/orchestrator.sh):
Replace the sed-pipe pattern with a temp-file pattern:
  ```
  _si_log=$(mktemp /tmp/spawn_instance_slot${slot}_XXXXXX.log)
  spawn_instance "$slot" ... >"$_si_log" 2>&1 || true
  sed 's/^/[orchestrator] /' < "$_si_log" >&2
  rm -f "$_si_log"
  ```
Now: spawn_instance runs with NO pipe. When it returns, the orchestrator immediately
proceeds to `sleep 3; _reflow_layout` and then enters `_read_fifo_msg`. The watchdog
has already written SLOT_DIED by then (within 2s of bwrap dying). Orchestrator reads
it, calls teardown_instance, sets active=false. T1.3 should pass.

### User feedback
User noted sessions were spinning too long on the pipe issue. Redirected toward
the correct architectural fix (temp file, no pipe) rather than continuing to
patch the bwrap side.

### Commits this session
- 375693b  fix(orchestrator): close pipe write-end in bwrap background process
- 1b06f5c  fix(orchestrator): use temp file for spawn_instance output, not a pipe

### SECOND FIX: Background spawn_instance (e447a37)

The temp-file fix (1b06f5c) didn't solve the problem. spawn_instance ITSELF was
blocking the orchestrator for up to 3 minutes:
- _poll_for_java: up to 60s

- 57 stray `minecraftSplitscreen.sh testDirect all` bash procs (all ~48 min old,

### Root cause

  grandchildren blocked on the FIFO orphaned to init and accumulated.

### Fixes
- `modules/orchestrator.sh`: `_read_fifo_msg` opens read-write (`<>`) so open()
  returns immediately and `-t` governs the wait.
- `minecraftSplitscreen.sh` testDirect: recursive `_kill_tree` (TERM→KILL) +
  remove FIFO last in `_td_cleanup`, used by both the EXIT trap and post-test.
