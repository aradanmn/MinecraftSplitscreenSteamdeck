Continued from 2026-06-20 session (context ran out). Working on Steam Deck external display reliability via dock.

### Issues addressed

**1. Stale WAYLAND_DISPLAY crash loop (both screens black, 2026-06-20 ~04:48)**
- Deck auto-suspended while docked for ~5.5h
- On resume, stale `WAYLAND_DISPLAY=wayland-1` left in systemd --user env (leaked from nested-KWin work)
- gamescope forced into nested backend → "Failed to connect to wayland socket: wayland-1" → crash loop
- 149 failures, both screens black, unrecoverable without manual intervention
- Fix: `systemctl --user unset-environment WAYLAND_DISPLAY` + sddm restart
- Reported to Valve: ValveSoftware/SteamOS#2467 (comment: start-gamescope-session clears DISPLAY/XAUTHORITY but not WAYLAND_DISPLAY)
- Our code was NOT the cause (resume watcher showed no entries, never fired)

**2. Resume watcher never fired**
- Old approach: persistent service tailing `journalctl -f -k` for "PM: suspend exit"
- Problem: journalctl -f process gets frozen during suspend, misses resume event
- Discovery: SteamOS 3.8.10 DOES use systemd-suspend.service (corrects old assumption that SteamOS bypasses systemd-sleep)
- Ships own hooks in /usr/lib/systemd/system-sleep/

**3. Steam not rendering to external on dock connect (2026-06-21)**
- gamescope correctly detected DP-1 (Sceptre O34, 3440×1440)
- But Steam wasn't rendering to it despite valid EDID
- Root: old hotplug script only restarted on missing EDID; Steam-not-rendering also needs restart
- User feedback: "to me it's all the same issue — Steam doesn't behave well with external displays"

### Changes made

**New: system/deck-display-fixes/ in repo**

`dock-display-hotplug.sh` — now restarts gamescope on ANY DP connect event (not just missing EDID). 60s stamp prevents rapid-fire. Env-sanitize before every restart.

`deck-display-resume.sh` — /etc/systemd/system-sleep/ hook, fires post-resume, starts oneshot worker --no-block

`resume-display-restore.sh` — oneshot worker: sleep 8 → sanitize env → restart gamescope if docked. Logs to /tmp/resume-display-restore.log. RESUME_RESTORE_DRYRUN=1 for testing.

`resume-display-restore.service` — Type=oneshot, on-demand (replaces Restart=always journalctl-tail)

`install.sh` — deploys all files to Deck

### Deployed to Deck

### Key finding
SteamOS 3.8.10 build 20260617.1 uses systemd-sleep (not /sys/power/state direct write). The /etc/systemd/system-sleep/ hook path is the robust resume trigger.
