# Session Log — 2026-06-17: Gamescope Windowing & TinyWM Integration

## Context
Continued work on Minecraft Splitscreen for Steam Deck. Previous sessions established:
- Game Mode (gamescope) ignores xdotool window positioning
- `:1` second Xwayland from `--xwayland-count 2` exists but invisible
- Controller isolation via bwrap + `--bind /dev/null` masking
- Need a proper window manager inside gamescope

## What Was Tried

### 1. Override Redirect + Unmap/Remap Hack
Modified `instance_lifecycle.sh` and `window_manager.sh` to:
- Set `override_redirect` on Minecraft windows immediately on discovery
- Unmap → set `override_redirect` → move/resize → remap cycle
- Attempted to bypass KWin's window management

**Result:** Failed — KWin in `--x11-display` mode still tiles windows. The override_redirect unmap/remap didn't stick in gamescope's XWayland.

### 2. Xephyr
Considered installing Xephyr for a nested X server with standard WM.

**Result:** Xephyr not available on SteamOS, would need `sudo pacman -S xorg-server-xephyr` which requires disabling SteamOS read-only mode. Deferred.

### 3. TinyWM (First Attempt — Test Script on Deck)
Wrote `/tmp/tinywm.py` — 146-line Python ctypes minimal X11 WM.
- Ran on `:1` (gamescope's second Xwayland)
- Launched two GTK test windows (SplitscreenLEFT, SplitscreenRIGHT)
- Used `show_msg()` GTK popups for communication

**Result:** No windows visible. The `:1` display is a separate X server not composited by gamescope onto the visible output. TinyWM needed to run on `:0`.

### 4. TinyWM (Second Attempt — Integrated via Subagent)
Dedicated coding subagent (deepseek-chat) rewrote the integration:
- **`modules/tinywm.py`** (539 lines) — Full X11 WM using ctypes, runs on `:0`
- Reads `splitscreen_state.json` for grid layout geometry
- `start_tinywm()` / `stop_tinywm()` / `sync_apply_layout()` in window_manager.sh
- `nested` handler in orchestrator (matches Steam's LaunchOptions= "nested")
- Added `--tinywm-test` developer mode for visual verification

## Key Technical Insights
1. **gamescope's XWayland is not a standard X11 WM** — SubstructureRedirect can be registered but compositing behavior differs
2. **`:1` is invisible** — gamescope creates it for nested compositors but doesn't display it
3. **`:0` with TinyWM** — TinyWM registers as WM on `:0` via SubstructureRedirectMask, intercepting MapRequest/ConfigureRequest events
4. **Steam shortcut passes `nested`** as LaunchOptions — script must handle unknown args by mapping them to desired behavior
5. **LD_PRELOAD issues** — Steam's `gameoverlayrenderer.so` (32-bit) spews harmless warnings in all subprocesses
