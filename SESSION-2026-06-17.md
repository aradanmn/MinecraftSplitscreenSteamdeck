### Research: Gamescope Compositing
Nemotron researched gamescope's source code:
- gamescope is a single-game-window compositor
- Uses `STEAM_GAME` atom to identify game windows
- xdotool geometry flows through XWayland but gamescope resizes focused window to fullscreen
- **Xephyr** is the recommended approach for docked gamescope multi-window mode
- Research doc written to `RESEARCH-GAMESCOPE-WINDOWING.md`

### 3-Round Challenge/Refine on the Fixes
Nemotron challenged every fix before applying:

**Round 1 — xdotool argument order:**
- Challenge: Does xdotool set_window care about argument order?
- Research: xdotool uses POSIX getopt with `+` prefix. Options MUST come before positional args (WID).
- Result: The "fix" that put WID before --name was WRONG. **Reverted.**

**Round 2 — SDL_LINUX_JOYSTICK_CLASSIC:**
- Challenge: Do we actually need this flag? Controlify uses SDL3, not SDL2.
- Research: The env var name for SDL3 is `SDL_JOYSTICK_LINUX_CLASSIC` (different from SDL2's `SDL_LINUX_JOYSTICK_CLASSIC`)
- Result: Both are harmless. `SDL_JOYSTICK_DEVICE` is already set, so neither flag matters functionally.

**Round 3 — Xephyr for docked gamescope:**
- Challenge: Can we avoid a new dependency?
- Research: Alternatives (X11 reparenting, bwrap with separate display) all require reimplementing Xephyr
- Result: Xephyr confirmed as correct recommendation.
