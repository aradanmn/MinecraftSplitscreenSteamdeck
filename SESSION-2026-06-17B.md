## Problem Statement

Nested KWin inside gamescope was drawing two windows, but both appeared stacked
at position (0,0). The goal: red window fills top half, blue fills bottom half.

Three approaches had already been ruled out before this session:
- **xdotool in gamescope's XWayland directly** ‚Üí gamescope ignores ConfigureRequest
- **KWin window rules (kwinrulesrc)** ‚Üí format uncertain for KDE 6, didn't work
- **wmctrl** ‚Üí not installed on Steam Deck

---

## Debugging Steps

### Step 1: Discovered wrong Steam shortcut argument

The Steam shortcut was passing `diagnose` as the launch argument. Script expected
`nested`. Fixed by removing argument requirement entirely.

**Fix:** Auto-detect context from environment variables instead of argument:
```bash
if [[ "${XDG_SESSION_DESKTOP:-}" == "KDE" || "${XDG_CURRENT_DESKTOP:-}" == "KDE" ]]; then
    launchWindowTest   # already inside nested KWin
else
    nestedPlasma       # in gamescope, start KWin
fi
```

Inside nested KDE session: `XDG_SESSION_DESKTOP=gamescope` but
`XDG_CURRENT_DESKTOP=KDE` ‚Äî second condition catches it correctly.

### Step 2: Added debug logging

Added `exec 2>>"$LOG"` + `set -x` to capture every step to
`/tmp/splitscreen-debug.log`. User launches in Game Mode, I read via SSH.

### Step 3: Full Plasma desktop loading

`startplasma-wayland` starts plasmashell alongside KWin, loading the full desktop
with taskbar and icons. We only want KWin as a bare window manager.

**Fix:** `pkill plasmashell` at the start of `launchWindowTest()`.

### Step 4: wmctrl not found, xdotool is available

Debug log showed `+ command -v wmctrl` returning nothing. The wmctrl repositioning
loop was silently skipped.

**Fix:** Switched to `xdotool search --pid $PID` for repositioning.

But this also returned empty: `WID1= WID2=`.

### Step 5: ROOT CAUSE ‚Äî GTK using Wayland backend

Inside the nested KDE Plasma/KWin session, `WAYLAND_DISPLAY` is set. GTK3
defaults to the **Wayland** backend when a Wayland display is available.

Consequences:
- `win.move(0, 540)` ‚Üí **no-op** (Wayland clients cannot self-position; the
  compositor decides window placement)
- `gdkwin.set_override_redirect(True)` ‚Üí **no-op** (X11-only concept, ignored
  by GDK Wayland backend)
- `xdotool search --pid` ‚Üí **empty** (xdotool is X11-only; Wayland surfaces are
  invisible to it)

This is why both windows appeared at the same position despite the Python code
explicitly calling `win.move(0, 0)` and `win.move(0, 540)`.

**Fix:** Force GTK to use XWayland (the nested KWin's X11 display on `:2`):
```bash
export GDK_BACKEND=x11
```

This one line was the entire fix.

## Final Working Solution

### Environment
- Gamescope (Game Mode) outer compositor
- `startplasma-wayland` ‚Üí nested KWin Wayland compositor inside gamescope
- KWin's XWayland provides `DISPLAY=:2`

### minecraftSplitscreen.sh flow

**On gamescope launch (Steam shortcut, no args):**
1. `nestedPlasma()` runs
2. `kwriteconfig6` disables KDE 6 auto-tiling
3. `kwinrulesrc` written with forced positions (belt-and-suspenders, may or may
   not help but doesn't hurt)
4. `~/.config/autostart/splitscreen-test.desktop` written pointing back to
   the same script
5. `exec dbus-run-session startplasma-wayland` ‚Äî never returns

**KDE autostart fires the script again inside the new session:**
1. `XDG_CURRENT_DESKTOP=KDE` ‚Üí `launchWindowTest()` runs
2. `pkill plasmashell` ‚Äî removes taskbar/desktop icons, keeps KWin
3. `export GDK_BACKEND=x11` ‚Äî **the critical fix**
4. P1 python3: `realize()` ‚Üí `set_override_redirect(True)` ‚Üí `move(0, 0)` ‚Üí `show_all()`
5. P2 python3: `realize()` ‚Üí `set_override_redirect(True)` ‚Üí `move(0, HALF_H)` ‚Üí `show_all()`
6. `xdotool search --pid` + `xdotool windowmove/windowsize` loop as backup

### Result

‚úÖ **CONFIRMED WORKING** ‚Äî red window top half, blue window bottom half,
clean screen with no desktop UI.

## Key Technical Findings

| Finding | Detail |
|---------|--------|
| GTK backend inside nested KDE | Wayland by default ‚Äî `win.move()` is a no-op |
| Fix | `GDK_BACKEND=x11` routes GTK through XWayland |
| override_redirect | Works in XWayland; bypasses KWin placement entirely |
| xdotool | Works on `DISPLAY=:2` (KWin's XWayland) ‚Äî can find by PID, move, resize |
| wmctrl | NOT installed on Steam Deck |
| plasmashell | Must be killed to get bare KWin surface |
| Auto-detect inside nested session | `XDG_CURRENT_DESKTOP=KDE` (even though `XDG_SESSION_DESKTOP=gamescope`) |
| DISPLAY inside nested session | `:2` (not `:1`) |
