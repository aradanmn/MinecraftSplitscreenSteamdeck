## Gate evaluation

- **4P completes, all instances alive, no OOM, no SLOT_DIED** — PASS both phases,
  every cycle of the campaign (journal + session logs clean).
- **rss_sum_max ≤ 12288 / memavail_min ≥ 1024** — PASS with room (7.8GB / 2.1GB).
- **FPS ≥ baseline −5% per screen per N** — PASS everywhere; branch leads +25% p50
  and ~+40% 1%-lows at 4P, ~2x flight lows at 2P; parity at 1P/3P.
- **swap Δ < 256/cycle** — breached by BOTH phases (297 vs 335) in the S3 flush after
  the quad slam; branch swaps LESS than baseline. Ruled protocol-attributable
  (documented maintainer call per runbook).
- **Soft gates** — smoothness 4/5 both phases; stutter reports equal or better on
  branch; temps ≤72°C.

## The headline correction

v1's baseline "memory wall" (1.4GB swap, PSI 5.16 at 4P) did NOT reproduce under
v2 on either install — it was substantially protocol-borne (sustained multi-front
creative flight + night mob spawning). The branch's true value: higher FPS floors
and steady-state throughput at 3–4P, faster slam recovery, ~350–840MB less RSS at
1–3P (FerriteCore), and equal-or-better memory behavior at 4P.
