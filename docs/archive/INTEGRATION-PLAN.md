## 1. End-user flow today (install ŌåÆ run)

### Install ŌĆö `install-minecraft-splitscreen.sh`
1. Downloads modules to a temp dir from `REPO_BASE_URL` = `ŌĆ”/main/modules` (or copies a local `modules/` if run from a clone).

## 3. Integration plan

> **CORRECTION (user, 2026-06-22):** `minecraftSplitscreen.sh` is meant to be **generated by the installer**, not shipped hand-written. The repo-root `minecraftSplitscreen.sh` is our rapid-testing copy; production is supposed to GENERATE it via `launcher_script_generator.sh`. This reframes items A and the former "delete the generator" line below.

### A0. BLOCKER ŌĆö pick ONE launcher source of truth (the root cause)
There are currently **two** drifted launcher implementations:
- `modules/launcher_script_generator.sh` ŌĆö the *intended production generator* (heredoc template + `__PLACEHOLDER__`/sed). **Stale (v2.1.1, Feb 1):** sources only 6 runtime modules (**missing `dex.sh`**), runs the OLD `launchGames`/`nestedPlasma` static flow, and is **orphaned** ŌĆö not in the installer's module lists, never sourced or called.
- repo-root `minecraftSplitscreen.sh` ŌĆö the current architecture (sources all 7 modules incl. `dex.sh`, auto-detects config, new orchestrator/test paths). The installer currently deploys THIS via `setup_splitscreen_launcher_script` (a copy), not the generator.

**DECISION (user, 2026-06-22): Option B + deploy-time version stamping.**
- Keep `setup_splitscreen_launcher_script` (deploy the hand-written modular script, which auto-detects launcher config at runtime ŌåÆ portable across distros). **Delete `launcher_script_generator.sh`** (retired by deliberate decision, not because it was dead).
- **Add version stamping without generation** (decoupled): put `MCSS_VERSION`/`MCSS_COMMIT`/`MCSS_BUILD_DATE="__ŌĆ”__"` placeholders near the top of the script (with `dev`/`unknown` defaults), and `sed`-stamp them in `setup_splitscreen_launcher_script` right after the `cp`:
  ```bash
  commit=$(git -C "$SCRIPT_DIR" rev-parse --short HEAD 2>/dev/null || echo unknown)
  ver=$(cat "$SCRIPT_DIR/VERSION" 2>/dev/null || echo dev)
  sed -i -e "s/__MCSS_VERSION__/$ver/" -e "s/__MCSS_COMMIT__/$commit/" \
         -e "s|__MCSS_BUILD_DATE__|$(date -Iseconds)|" "$launcher_script"
  ```
  Expose it via a startup log line and a `--version` flag. ~6 lines, single source of truth, no template/drift.
- (Rejected Option A ŌĆö restore generation: would re-introduce a second copy to keep in sync; not worth it now that config is auto-detected. The only A-only benefit, deterministic launcher choice when both PolyMC+PrismLauncher exist, is covered by logging the detected launcher + the `LAUNCHER_EXEC` env override.)

### A1. BLOCKER ŌĆö wire the production launch flow (`launchFromPlasma`)

- Fix: factor the nested-Plasma session setup (currently in `testPlasma`/`launchTestFromPlasma`) into a shared helper; `launchFromPlasma` runs it with `main()` inside (production), `test` runs it with the harness inside.
- Without this, a real user launching from Steam gets no working splitscreen. Single most important functional item.

### Preflight sketch
```bash
# Verify required tools; fail fast with a clear message instead of a cryptic
# mid-launch crash. Critical = launcher cannot function; session = nested-Plasma
# windowing. (xdotool/wlr-randr are intentionally omitted ŌĆö fallback-only.)
_preflight_deps() {
    local missing=()
    for bin in jq python3 bwrap dbus-run-session \
               kwin_wayland startplasma-wayland kscreen-doctor xdpyinfo; do
        command -v "$bin" >/dev/null 2>&1 || missing+=("$bin")
    done
    if (( ${#missing[@]} )); then
        echo "[mcss] Missing required tools: ${missing[*]}" >&2
        echo "[mcss] SteamOS: sudo steamos-readonly disable && sudo pacman -S <pkg>; then retry." >&2
        return 1
    fi
    return 0
}
```
Most of these ship with a stock SteamOS desktop, so the practical value is a clear diagnostic on a non-standard image and an explicit, documented requirement set.
