> A follow-up audit (`BUG-AUDIT-2026-06-24-followup.md`) verified this doc against the code and

Multi-agent audit (8 module auditors → per-finding adversarial verification → dedupe).
**72 raw findings → 52 confirmed → 27 distinct: 2 Critical, 14 High, 8 Medium, 3 Low.**

**Top 3 to fix first:** (1) CONTROLLER_ADD field-parse corruption (js_node gets the vendor/
product appended); (2) controller_monitor udevadm pipe-subshell — `prev_nodes` never
persists, so device-change detection silently fails after the first event; (3) dex.sh Python
`args[]` IndexError family — six unguarded handlers crash the backend on short arg lists.

## CRITICAL
- **C1 — `orchestrator.sh:140-143` + `controller_monitor.sh:520-521,577` — CONTROLLER_ADD
  format mismatch (js_node corrupted).** Monitor emits 4 fields (`event js vendor product`);
  orchestrator's `${msg_arg#* }` strips only the first space → `js_node="/dev/input/jsX 28de
  11ff"`, passed straight to spawn_instance. (= the production bug; reported 5×.)
  FIX: `read -r event_node js_node phys_vendor phys_product <<< "$msg_arg"`; fix the stale
  2-field comment at orchestrator.sh:138.

## HIGH
- **H1 — `controller_monitor.sh:585` — udevadm monitor runs in a pipe subshell; `prev_nodes`
  never persists.** `udevadm monitor … | while read …` → loop is a subshell; assignments at
  ~610 are lost each iteration → device-change detection unreliable after the first event.
  FIX: process substitution — `while …; do …; done < <("$udevadm_cmd" monitor …)`.
- **H2 — `dex.sh:267,281,259,471,425,437` — `action_*` unpack `args[]` with no bounds check.**
  6 handlers (move_resize, move_resize_force, resize, spawn_placeholder, set_fullscreen,
  set_skip_taskbar) → IndexError on a short call. FIX: `if len(args) < N: return` guard.
- **H3 — `dex.sh:454-455` — `action_get_active_wid` checks `len>=4` but reads 8 bytes**
  (`prop[:8]` with `<Q`). FIX: check `len(prop) >= 8`.
- **H4 — `window_manager.sh:159,163` — `_verify_window_geometry` clobbers `ah` with a
  redundant 2nd query.** Pairs stale `aw` with fresh `ah` → inverts the match. (Reported 3×.)
  FIX: delete the second query; the first read captured all 4 atomically.
- **H5 — `window_manager.sh:159` — geometry read can leave vars empty → numeric-compare
  crash** (`[[ $ax -ne $ex ]]` with empty operand). FIX: validate non-empty before compare.
- **H6 — `watchdog.sh:77` + `dock_detection.sh:243,255` — FIFO writes unprotected under
  `set -e`.** A broken pipe (orchestrator closed the read-end) exits the whole monitor →
  orphaned slot-death / display-change detection. FIX: `… >> "$fifo" || true`.
- **H7 — `kwin_positioner.sh:87-89` — unvalidated `pid` injected into KWin JS as a bare
  identifier.** Malformed spec → `pid:abc` (a variable ref) → matches nothing silently.
  FIX: validate pid/x/y/w/h are integers, `continue` on failure.
- **H8 — `kwin_positioner.sh:136` — `tr -dc '0-9'` masks loadScript errors** (`ERROR: 123` →
  `123`). FIX: validate the whole string: `[[ "$id" =~ ^[0-9]+$ ]]`.
- **H9 — `orchestrator.sh:305-308` — `handheld_flow` ignores the FIFO read timeout; no
  monitor heartbeat.** Loops forever even if controller_monitor/watchdog died. FIX: periodic
  `kill -0` aliveness checks → cleanup + return on death.
- **H10 — `orchestrator.sh:159-167` — backgrounded reflow swallows `_reflow_layout` failure**
  (`{ …; _reflow_layout; } &` + `|| true`). FIX: log on failure + a `reflow_needed` retry flag.
- **H11 — `instance_lifecycle.sh:230-232,256` — hardcoded `max_iterations=120`/`240` duplicate
  the poll-timeout constant.** FIX: derive from `POLL_TIMEOUT_S` / `POLL_INTERVAL_S`.
- **H12 — `instance_creation.sh:185` — `local preserve_options_txt=false` shadows the value
  from line 80** → wrong value passed to install_fabric_and_mods (line 201). FIX: drop the
  re-declaration.
- **H13 — `instance_creation.sh:369,379,386,399,408` (+ mod_management) — `.files[0]` vs
  `.files[] | select(.primary==true)`.** Grabs a non-primary file → wrong mod downloaded /
  false MISSING. FIX: standardize on the primary selector; guard empty `.files`.
- **H14 — `dock_detection.sh:148` — wrong awk field** (`$2` is the output index, name is `$3`).
  FIX: `awk '{print $3}'`.

## MEDIUM
- **M1 — `orchestrator.sh:161,189` + `instance_lifecycle.sh:161` — `mktemp` without error
  check** (empty path on full /tmp). FIX: `… || { echo ERROR; return 1; }`.
- **M2 — `orchestrator.sh:159-167` — backgrounded brace-group leaks its temp log on signal**
  (invisible to parent cleanup). FIX: per-subshell `trap rm` or per-session TMPDIR.
- **M3 — `orchestrator.sh:552` — `sync_apply_layout` called with empty W/H** → re-query may
  return stale resolution after a hotplug. FIX: pass actual resolution / error-checked re-query.
- **M4 — `orchestrator.sh:102-103` — fragile `xdpyinfo` dimension parse** (`awk $2 | cut`).
  FIX: `sed -n 's/.*dimensions:[[:space:]]*\([0-9]*\)x.*/\1/p'`.
- **M5 — `controller_monitor.sh:453-461` — `get_controller_by_index` doesn't validate index**
  for `sed -n "${index}p"`. FIX: guard `[[ "$index" =~ ^[1-4]$ ]]`.
- **M6 — `controller_monitor.sh:509` — debounce uses `<` not `<=`** (dup exactly at 500ms not
  debounced). FIX: `<=`.
- **M7 — `dex.sh:37,558-560,619` — `/tmp/dex_$$.py` leaks on crash** (cleanup not EXIT-trapped
  by design). FIX: manual cleanup at orchestrator exit, or `$XDG_RUNTIME_DIR`.
- **M8 — robustness cluster (`instance_creation.sh:71,205`, `version_management.sh:34-35`,
  mod_management timeouts)** — coarse `set +e` windows mask errors; `[0:20]` version slice
  assumes ≥20; inconsistent curl/wget timeouts. FIX: per-command error handling; validate
  version count ≥5; `TIMEOUT_SHORT/MED/LONG` constants.

## LOW
- **L1 — `watchdog.sh:26-89` — infinite loop, no signal trap** (must be force-killed).
  FIX: `trap 'exit 0' TERM INT`.
- **L2 — `orchestrator.sh:28-31` — dead constants** `ORCHESTRATOR_SETTLE_DELAY_S`/
  `IDLE_TIMEOUT_S`. FIX: remove or wire in.
- **L3 — hygiene cluster (`dex.sh:603-609,504`, `instance_lifecycle.sh:151-156`,
  `controller_monitor.sh:597`, `instance_creation.sh:348,360`)** — jq `${slot}` interpolation
  instead of `--arg`; `set_decorations` lacks `args[0]` guard; odd mask-controller arg dropped;
  magic `0.1` sleep; DEBUG_MODE temp files leak. FIX: jq `--arg`; arg guard; fail on unpaired
  mask arg; name the settle constant; clean debug temp files on EXIT.
