## Minor Items (Not Bugs)

### 1. spawn_instance backgrounded subshell exit codes

The backgrounded spawn runs in a subshell: `{ spawn_instance; } & _SPAWN_PIDS+=($!)`. The parent shell does NOT capture the subshell's exit code. This is intentional (non-blocking launch), and the actual error handling happens INSIDE the subshell (spawn failure clears the slot reservation).

**Status:** By design; watchdog and window-gone detection provide independent safety nets.

---

### 2. update_slot_state subshell failures

When jq or flock fails inside `update_slot_state`, an `exit 1` happens in the flock subshell. The parent (`_handle_msg`, etc.) does not check the return value. But: the state write either succeeds or doesn't; subsequent readers return empty/null; watchdog notices. Errors are logged.

**Status:** Acceptable; the atomicity is preserved, and callers tolerate missing state gracefully.

---

### 3. Device node reuse on reconnect (KNOWN LIMITATION)

**From orchestrator.sh:1062–1080 (CONTROLLER_REMOVE comment):**

> HONEST DISCLOSURE (pre-existing defect, UNCHANGED here, OUT OF SCOPE): this deliberate no-op means a reused-eventN fast-flap, or a jsN that vanishes with NO udev remove (driver crash / USB autosuspend), leaves a STILL-ALIVE zombie slot...

This is a documented, accepted design limitation. The fix (re-launch the same slot on detect replacement) is deferred. No code change recommended.

---

### 4. spawn_instance slot pre-reservation edge case

The slot is marked `active: true` BEFORE spawn runs, to prevent double-allocation on rapid CONTROLLER_ADD (N2 comment, line ~940). If spawn fails in the backgrounded subshell, cleanup runs and clears the reservation. If that cleanup fails, the slot remains `active: true` with `bwrap_pid: null`.

`_reap_dead_slots` then skips it: `[[ -z "$bwrap_pid" ]] && continue` (lines ~484). The watchdog cannot detect this either (no real process to kill).

**Risk:** Very low. The scenario requires:
1. spawn_instance to fail (bwrap missing/bad permissions)
2. Its cleanup to also fail
3. No window detection in >4s

**Current state:** If window detection succeeds (most cases), `bwrap_pid` is set and slot becomes reap-able. Window-gone detection catches the rest after debounce.

**Fix if needed:** After spawn failure in the brace group (line ~941), unconditionally clear the slot. The code already does this.
